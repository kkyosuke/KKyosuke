Slackbot 導入設計書

概要 (Overview)

GitHub App (PR Review Agent) にSlackbotの機能を統合し、以下の要件を満たすシステムを構築します。

  1. 通知 (Notification): PRレビューの完了やエラー発生などの重要なイベントをSlackチャンネルに通知する。
  2. 操作 (Interaction): Slackのコマンドやメンションから、PRレビューの再実行や状態確認を行えるようにする。

動作環境・制約

本システムは Cloudflare Workers 上で動作する Hono アプリケーションです。
そのため、Node.js環境に強く依存する @slack/bolt フレームワークは使用せず、以下の構成で実装を行います。

  • Web API: @slack/web-api パッケージを利用して、Slackへのメッセージ送信を行います。
  • Events/Interactivity API: Hono内に専用のエンドポイント (POST /webhook/slack) を作成し、Slackからのリクエスト（コマンドやメンション）を受信・処理します。

アーキテクチャ構成 (Architecture)

1. ディレクトリ構成の追加・変更箇所

src/
 ├── handlers/
 │    └── slack.ts         # SlackからのWebhookを受け付けるHonoハンドラー
 ├── lib/
 │    └── slack/
 │         ├── client.ts   # @slack/web-api の初期化とラッパー関数
 │         └── verify.ts   # Slackからのリクエストの署名検証ロジック
 └── config/
      └── env.ts           # Slack用の環境変数定義を追加

2. 環境変数 (Environment Variables)

以下の環境変数をCloudflare Workers (および .env) に追加します。

  • SLACK_BOT_TOKEN: SlackアプリのBot User OAuth Token (xoxb- から始まる)
  • SLACK_SIGNING_SECRET: Slackからのリクエストの正当性を検証するためのシークレット
  • SLACK_NOTIFICATION_CHANNEL: 通知を送信するデフォルトのチャンネルID（オプション）

機能詳細とシーケンス

A. 通知機能（PRレビュー完了時の通知）

AIによるレビューが完了した際に、結果のサマリーをSlackへ送信します。

  1. src/jobs/review/agent.ts または src/handlers/webhook.ts の処理完了部分。
  2. slackClient.chat.postMessage を呼び出し、対象のPRリンク、レビューステータス（Approve / Request Changes など）、およびサマリーを通知。

シーケンスイメージ:

sequenceDiagram participant GitHub participant App as CF Workers (Hono) participant AI as Claude / Gemini participant Slack GitHub->>App: Webhook (pull_request.opened) App->>AI: PR Diff 解析 AI-->>App: レビュー結果 App->>GitHub: レビューコメント投稿 App->>Slack: 通知「PR #XX のレビューが完了しました」

B. インタラクティブ機能（Slash Command によるレビュー指示）

Slackのコマンド（例: /pr-review <PRのURL>）から任意のPRに対してレビューをトリガーします。

  1. ユーザーがSlackで /pr-review https://github.com/owner/repo/pull/123 を実行。
  2. Slackから POST /webhook/slack エンドポイントへリクエストが飛ぶ。
  3. verify.ts で SLACK_SIGNING_SECRET を用いてリクエストを検証。
  4. Honoハンドラがリクエストを受け付け、対象のPR情報をパース。
  5. Slackへ即座に「レビューを開始します...」と応答を返す。
  6. バックグラウンド処理（ctx.waitUntil または Queue）でレビュー処理を続行し、完了後に結果をSlackへスレッド返信。

シーケンスイメージ:

sequenceDiagram participant User participant Slack participant App as CF Workers (Hono) participant GitHub participant AI User->>Slack: /pr-review Slack->>App: POST /webhook/slack App-->>Slack: 200 OK "レビューを開始します" App->>GitHub: PR詳細取得 App->>AI: レビュー実行 AI-->>App: 結果 App->>GitHub: レビューコメント投稿 App->>Slack: "レビューが完了しました (リンク)"
