Slackbot 導入設計書

概要 (Overview)

GitHub App (PR Review Agent) にSlackbotの機能を統合し、以下の要件を満たすシステムを構築します。

  1. 通知 (Notification): PRレビューの完了やエラー発生などの重要なイベントをSlackチャンネルに通知する。
  2. 操作 (Interaction): Slackのコマンドやメンションから、PRレビューの再実行や状態確認を行えるようにする。

動作環境・制約

本システムは Cloudflare Workers 上で動作しています。
Node.js環境に強く依存する公式の @slack/bolt は使用せず、Cloudflare Workers 用に最適化されたBolt風フレームワーク slack-cloudflare-workers を採用します。

  • 書き心地はBoltと同じ: モーダルやアクション（Block Kit）などの複雑なUIも、Boltと全く同じ感覚で実装できます。
  • Honoとの共存: 既存のHonoのルーティング（GitHub Webhook用）と並行して、Slackからのリクエストを処理できます。

アーキテクチャ構成 (Architecture)

1. ディレクトリ構成の追加・変更箇所

src/
 ├── handlers/
 │    └── slack.ts         # slack-cloudflare-workers のアプリ初期化・ハンドラー定義
 └── config/
      └── env.ts           # Slack用の環境変数定義を追加

2. 環境変数 (Environment Variables)

以下の環境変数をCloudflare Workers (および .env) に追加します。

  • SLACK_BOT_TOKEN: SlackアプリのBot User OAuth Token (xoxb- から始まる)
  • SLACK_SIGNING_SECRET: Slackからのリクエストの正当性を検証するためのシークレット
  • SLACK_NOTIFICATION_CHANNEL: 通知を送信するデフォルトのチャンネルID（オプション）
