Slack Home Tab ITニュース表示機能 設計書

概要 (Overview)

SlackアプリのHome Tab（ホームタブ）の上部に、その日の注目ITニュースを1件表示します。ニュースは無料のニュースサイト（RSS）から取得した直近のIT関連ニュースから、AIがエンジニアにとって最も面白そうなものを1件選定します。

処理フロー (Processing Flow)

  1. Homeタブへのアクセス: ユーザーがSlackでHomeタブを開きます（app_home_opened イベント発生）。
  2. キャッシュの確認: Cloudflare Workers KV (AI_KYOSUKE_KV) に対して、その日（JST）のニュースデータがキャッシュされているか確認します。
  3. キャッシュが存在する場合: キャッシュされたニュースデータ（タイトル、URL、AIの推薦コメント）をそのまま返却します。
  4. キャッシュが存在しない場合:
    • https://news.yahoo.co.jp/rss/topics/it.xml から直近のITニュースを最大20件取得します。
    • 取得したニュース一覧をプロンプトに組み込み、Anthropic API（Claudeモデル）の generateObject を呼び出し、最もエンジニア向けの面白い記事を1件選定させます。同時に推薦理由（1〜2文）も生成させます。
    • 選定された結果をKVに有効期限1日 (expirationTtl: 86400) で保存します。
  5. UI構築と表示: Block Kitを用いてヘッダーとセクションブロックを生成し、Welcomeブロックの直下にニュースを表示します。

システム構成 (System Architecture)

  • RSS取得: fetch を用いて外部RSSをテキスト取得し、簡易正規表現にてパース（src/lib/news.ts）。
  • AIモデル: `@ai-sdk/anthropic` を利用。コストとパフォーマンスを考慮し、デフォルトのReportモデル（例：claude-haiku-4-5等）を使用。
  • KVストア: AI_KYOSUKE_KV に daily_it_news_YYYY-MM-DD のキー形式で保存。1日の期限付きでキャッシュするため、同日内の2回目以降のアクセスは高速にレスポンスされます。
注意点 (Note)
その日の最初にアクセスしたユーザーは、ニュースの取得とAIの呼び出しがリアルタイムに行われるため、Homeタブの読み込み（Blockのpublish）に数秒の遅延が生じます。Slackの要件上Viewの更新は非同期でも可能であるため、運用上の問題はありません。
