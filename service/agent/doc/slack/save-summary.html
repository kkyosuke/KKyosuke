スレッド保存機能 (save-summary) 仕様書

概要 (Overview)

Slackのメッセージショートカット、または特定キーワードのメンションを利用して、指定したスレッドでのやり取りをLLM（AI）で要約し、D1データベース（AI_KYOSUKE_DB）に「進捗情報」として保存する機能です。1つのスレッド内に複数人の投稿がある場合でも、個人ごとに内容を分けて要約・評価を行います。処理完了後、該当スレッドにボットが結果を返信します。

トリガー情報

  • パターンA: メッセージショートカット (Message Shortcut)
    • Callback ID: save-summary
    • UI表示名: スレッドを進捗として保存
  • パターンB: App Mention (Botへのメンション)
    • キーワード: @kyosuke.ai 日報をまとめて

主要な処理詳細

1. トリガーと即時応答

Slack APIの制約（3秒以内の応答必須）を回避するため、ショートカットの場合は即座にAckを返します。メンションの場合は、ユーザーに処理開始を伝えるためのメッセージをスレッドに即座に投稿します。

2. スレッド履歴の取得

対象のメッセージがスレッド内のリプライか、親メッセージかに関わらず適切に thread_ts を取得し、conversations.replies APIを用いてスレッド内の全てのやり取りを取得します。

3. テキストの整形と要約生成

LLMが文脈と各個人の発言を理解しやすいように、各メッセージを [投稿日時] <@ユーザーID>: メッセージ内容 の形式に結合します。LLMにはJSON形式で返却するよう指示し、参加している全ユーザーごとの「進捗（%）」「評価スコア（5点満点）」「要約テキスト」を一度の呼び出しで生成させます。

4. データの永続化 (D1 Database)

要約結果を、一意のID（UUID）と共にD1データベースの progress_summaries テーブルに保存します。スレッドに複数人が参加している場合は、人数分のレコードが作成されます。

  • id: UUID
  • user_id: 投稿した個人のSlack ID
  • target_date: 対象日（スレッド開始日時などから取得）
  • progress_percent: LLMが判定した進捗率（0-100）
  • evaluation_score: LLMが判定した評価スコア（1-5）
  • summary_text: LLMによる個人の要約テキスト
Note: スキーマ定義については別途 schema.sql 等を作成してマイグレーションを行う必要があります。
