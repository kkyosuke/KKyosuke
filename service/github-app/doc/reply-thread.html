インラインコメント（スレッド）返信時の処理フロー

このドキュメントでは、AIが投稿したインラインコメント（レビューのスレッド）に対して、開発者が返信を行った際の自動対応処理（ReplyAgent）についてまとめています。

1. 概要

開発者からの「対応しました」「質問があります」といった返信をトリガーとし、ボットがスレッドの会話内容（質問と回答のやり取り）と該当コードの差分を読み取ります。LLMがその内容を評価し、追加の返信が必要か、スレッドを解決（Resolve）してよいかを自律的に判断してアクションを実行します。

3. 処理の詳細ステップ

Step 1: スレッド内容の取得

Webhookで受け取った commentId を元に、Pull Requestの全スレッド情報から該当するスレッドを特定します。そのスレッド内に含まれる全てのコメント（これまでのやり取り）を時系列で取得し、結合してひとつの文脈データを作成します。

Step 2: LLMによる評価 (質問と回答の解析)

以下の情報をLLMに渡し、スレッドの現在の状態を評価させます：

  • スレッドの会話履歴: AIの初期指摘と、それに対する開発者の返信内容
  • 対象コードの差分: 指摘箇所周辺の最新のコード状態
  • リポジトリガイドライン: 存在する場合、プロジェクト固有のルール

Step 3: アクションの決定と反映

LLMは評価結果として、以下のアクションのいずれかを返します。ボットはこれに従い GitHub 上に反映させます。

  • REPLY: 開発者からの質問に対する回答や、修正が不十分な場合の追加指摘を返信します。
  • RESOLVE: 指摘事項が正しく修正されている、あるいは妥当な理由が説明されたと判断した場合、スレッドを「解決済み (Resolved)」にします。
  • REPLY_AND_RESOLVE: 最後に「確認しました、ありがとうございます」などの軽い返信を行いつつ、スレッドを「解決済み」にします。
  • IGNORE: ボットが返信する必要がない（開発者同士の会話など）と判断した場合は、何もしません（そのままにする）。
